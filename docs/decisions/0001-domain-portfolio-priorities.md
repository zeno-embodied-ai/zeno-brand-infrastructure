# ADR-0001：域名组合优先级与预算门禁

- 状态：`proposed`
- 日期：2026-07-12
- 决策范围：可注册主域组合，不包含 DNS 记录实施
- 关联文档：[`../domain-matrix.md`](../domain-matrix.md)

## 背景

Zeno 已知生产主域为 `zeno-ai.org`。为了降低主品牌仿冒、拼写混淆和产品入口分散风险，需要在不假设可注册性、不执行购买的前提下，为候选域名建立稳定的优先级与预算门禁。

## 决策

1. `zeno-ai.org` 定义为 P0，继续作为唯一 canonical 主域。
2. `zeno.ai`、`zeno-robotics.com` 和 `zeno-robotics.org` 定义为 P1，只有在可得性、价格、法务和安全条件通过后才提交优先购买审批。
3. `zeno.dev`、`zeno-embodied.ai` 和 `zeno-platform.ai` 定义为 P2；两个定位型 `.ai` 域名第一年最多选择一个。
4. `zeno-humanoid.com`、`zeno-humanoid.org` 和 `zeno-marine.com` 定义为 P3，默认不在第一年购买。
5. 第一年度建议核心预算为 USD 1800；加 P2 条件储备后的建议授权上限为 USD 2400。
6. 所有候选可注册性维持 `TBD`，直到获得明确采购评估授权。
7. 所有辅助域名只跳转至 `zeno-ai.org` 或其子域/路径，不建立第二个 canonical 网站。

## 理由

- 短主品牌和 `robotics` 近似名称具有最高的身份混淆和钓鱼风险。
- 产品专用域名会增加续费、证书、监控和退役成本，优先使用主域路径更利于长期治理。
- `.ai` 可能存在多年注册、premium 获取和 premium 续费风险，需要比通用 TLD 更严格的单域门禁。
- P0–P3 分级使 Brand、Legal、Security 和 Finance 可以按风险顺序审批，而不是一次性购买全部候选。

## 影响

### 正面影响

- 统一 canonical，降低品牌和 SEO 分散。
- 为高价值候选建立明确的停止价格，避免紧迫感驱动的超预算采购。
- Markdown 与 CSV 可以同步评审并用于后续资产治理。

### 代价与风险

- 未购买的候选可能被第三方持有；当前未查询，因此风险状态为未知。
- 预算上限可能因汇率、registry 政策和 premium 分类而失效，需要采购前复核。
- 防御性域名本身增加账号、续费、DNS 和恢复管理负担。

## 复审条件

出现以下情况时重新评审：主品牌或产品命名变化、候选出现真实混淆/钓鱼事件、产品正式发布、registry/registrar 政策显著变化、报价超过门禁或三年持有复审到期。

## 未授权操作

本决策不批准查询实时可注册性、购买、续费、转让、经纪、竞价、nameserver 变更、DNS/MX 修改或跳转上线。上述操作必须使用独立任务和明确人工批准。
