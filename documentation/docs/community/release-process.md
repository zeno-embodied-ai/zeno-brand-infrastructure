# Release Process

:::caution Draft / TBD
No Zeno product release channel, version convention, signing policy, artifact repository or support window is approved in this draft.
:::

## Intended release expectations

Every future release should identify its source revision, artifacts, compatibility scope, documentation version, validation record, upgrade notes and rollback path.

## To be confirmed

- Semantic versioning and pre-release policy.
- Artifact signing, provenance and publication process.
- Release notes, deprecation and support lifecycle policy.
